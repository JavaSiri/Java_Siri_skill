# 指标提取器

> **职责边界**：本 skill **只做一件事**——从需求文本中识别出指标并给出规范命名，输出一个 JSON 文件供后续 skill（`grill-with-docs` 等）使用。
> 不负责口径澄清、粒度指定、过滤条件、数据来源等任何后续阶段的内容。

## 工作流程

### Step 1：识别指标

扫描需求文本，识别业务指标：
- 有明确统计口径的度量（GMV、订单数、转化率、逾期率等）
- 带数量词的词组（"用户数"、"金额"、"笔数"、"率"、"比"）
- 报表中出现的统计字段

**忽略**：维度（省份、渠道、产品类型）、技术字段、通用词（日期、ID）。

### Step 2：命名（唯一关键动作）

**判断度量主语**：自问"我对什么计数 / 求和 / 取最早？"——答案就是度量主语。

**命名规则**：
- 技术名第一段必须是**度量主语**
- 格式：`{度量主语}_{动作}_{度量}`，例：`loan_application_num`、`credit_application_first_apply_date`
- **禁止**把粒度/范围/表名塞进技术名

**反例**：
- ❌ `credit_contract_loan_application_num`（把"授信合同"这个粒度当成主语塞进第一段）
- ✅ `loan_application_num`（主语是 `loan`）

### Step 3：输出

直接输出 JSON 并**写入物理文件**：

```json
{
  "metrics": [
    { "name": "credit_application_count", "display_name": "授信申请数", "status": "pending" },
    { "name": "loan_application_count", "display_name": "支用申请数", "status": "pending" }
  ]
}
```

- `name`：技术名，必须以度量主语为前缀
- `display_name`：中文展示名，**禁止裸词**，必须体现度量主语
- `status`：初始值固定为 `"pending"`

**写入路径**：`.cursor/knowledge/requirement/{YYYYMMDD}_{需求摘要拼音首字母}_metrics.json`
示例：`20260622_sxhtzh_metrics.json`

### Step 4：异常处理

> **核心原则**：遇到意外情况时，**先问用户，再决定如何处理**。禁止在用户不知情的情况下自行判断并输出结果。

#### 情况一：无法识别任何指标

- **判断标准**：需求文本中完全没有明确的统计度量词（GMV、笔数、金额、率、比等）
- **处理流程**：
  1. **先问用户**：告知"我无法从需求中识别出明确的统计指标"
  2. 由用户决定：
     - 用户补充指标 → 重新进入 Step 1
     - 用户确认输出空集合 → 输出 `{ "metrics": [] }` 并物理落盘
     - 用户说"跳过" → 告知用户跳过步骤的后果，由用户显式确认
- **禁止**：未经用户同意直接输出 `{ "metrics": [] }`

#### 情况二：指标名称不确定

- **判断标准**：能识别出指标，但命名存在歧义
- **处理流程**：
  1. **先问用户**：提出最贴近的命名方案，说明歧义所在
  2. 由用户确认或修正命名
  3. 用户确认后再落盘

#### 情况三：用户未提供需求

- **判断标准**：对话中没有业务需求文本
- **处理流程**：
  1. **先问用户**：请求输入需求内容
  2. 获得有效需求后再继续

#### 情况四：遇到其他意外情况

- **判断标准**：超出上述三种情况之外的任何不确定性
- **处理流程**：
  1. **先停下来**：说明当前遇到的问题
  2. **先问用户**：描述选项及各自影响
  3. 由用户决策后继续

> **通用纪律**：所有异常情况一律先问用户，禁止在用户不知情的情况下擅作主张。问用户是标准动作，不是"例外情况"。

## 示例

**输入**：
> 我要看授信申请数、支用申请数、以及授信通过数

**输出**：

```json
{
  "metrics": [
    { "name": "credit_application_count", "display_name": "授信申请数", "status": "pending" },
    { "name": "loan_application_count", "display_name": "支用申请数", "status": "pending" },
    { "name": "credit_pass_count", "display_name": "授信通过数", "status": "pending" }
  ]
}
```

文件：`.cursor/knowledge/requirement/20260612_sjsxzxjs_metrics.json`
