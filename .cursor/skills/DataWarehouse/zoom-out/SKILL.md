# Zoom Out — 数据仓库全局视图

我不熟悉这个数据域。拉远镜头，用数仓分层词汇（ODS / DWD / DWS / ADS），给我一张相关表、数据流向和依赖关系图。

## 输出结构

按以下顺序输出，简洁清晰：

### 1. 主题域定位

一句话说明这个表 / 指标属于哪个主题域，以及它在业务链路中的角色。

### 2. 分层血缘图

用 Mermaid 格式画出上下游关系，层级从下往上（ODS → DWD → DWS → ADS）：

```
ODS层  →  DWD层  →  DWS层  →  ADS层
                ↑
           [本表 / 本指标]
```

列出每一层的关键关联表，以及本表在其中的位置。

### 3. 上游溯源链

追溯到 ODS 层（或更远的业务源系统），列出每个关键节点的：
- 表名及所属层级
- 核心口径（聚合方式 / 去重逻辑 / 日期归因规则）
- 关键字段（JOIN 键、分区键、业务主键）

### 4. 下游消费链

本表被哪些下游表 / 报表消费，各自的用途是什么。

### 5. 口径与约束

- 该表的分区策略（按天 / 按月 / 不分区）
- 数据时效（增量 / 全量 / CDC）
- 已知的数据质量风险或特殊处理逻辑

### 6. 领域词汇对照

用项目自身的业务术语而非技术术语标注关键概念。

## 输出原则

- **只输出你实际探索到的信息**。不确定的地方明确标注，而不是猜测
- **优先引用数据字典或血缘系统**（如 Apache Atlas、Datahub、Metabase），其次才是代码反推
- 如果项目缺少元数据系统，标注这一事实，并说明哪些信息需要人工确认
