# 需求分析

## 定位说明

> **快速澄清工具，不是需求入口。**
>
> - 不产出持久化文件
> - 不替代 `to-prd`（PRD 是唯一需求入口）
> - 使用场景：需求信息不足、需要快速澄清后再走 to-prd
> - 流程：快速分析 → 澄清缺口 → 引导进入 to-prd

## Quick start

1. 读取需求文本
2. 按 5 个维度结构化分析
3. 输出分析结论
4. 识别口径缺口并引导澄清
5. **引导进入 to-prd**（需求入口）

## 分析维度

对需求进行以下结构化分析：

| 维度 | 内容 |
|------|------|
| **业务问题** | 还原用户真正想回答的问题，拆解核心指标 |
| **数据来源** | 识别表层级（DWD / DWS / ADS）和具体表名 |
| **口径确认** | 识别需人工确认的关键点（分子/分母、时间、样本口径） |
| **统计视角** | 「切片累积型」或「自然周期型」→ 见 classifiers/stat-view.mdc |
| **时间维度** | 「日维 / 周维 / 月维」→ 见 classifiers/time-dimension.mdc |

## 输出格式

```
需求分析结论
═══════════════════════════════════════

【业务问题】
  还原的业务问题：[一句话描述]
  核心指标：
    - [指标名]：[业务含义]

【分类结果】
  统计视角：[切片累积型 / 自然周期型]
  切片维度：[字段名 / 无]
  时间维度：[日维 / 周维 / 月维]（仅自然周期型填写）

【数据来源】
  数据层级：[DWD / DWS / ADS]
  涉及表：[表名列表]
  数据是否已加工：[是/否]

【口径确认】
  待确认项：
    1. [口径点] → [待确认]
    2. [口径点] → [待确认]

【引导结论】
  是否进入 to-prd：[是/否]
    - 是：输出「建议通过 to-prd 创建正式 PRD」
    - 否：输出「建议补充 xxx 信息后再进入 to-prd」
```

## 口径确认参考

| 确认项 | 典型问题 |
|--------|---------|
| 分子/分母口径 | 逾期笔数包含哪些状态？展期是否计入？ |
| 时间口径 | 还款截止到什么时间点？宽限期内还款算当期还是逾期？ |
| 样本口径 | 是否排除欺诈/提前结清样本？ |
| 指标定义 | "通过"的标准是什么？预审通过还是终审通过？ |
