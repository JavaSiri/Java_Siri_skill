## 关键口径说明

### 粒度说明

每行代表一笔授信申请在某一节点的执行记录。一次授信申请可能经过多个节点，因此 `credit_sk + node_id` 构成一条记录。

### 节点耗时计算

`duration_seconds = end_time - start_time`，反映决策引擎中该节点的执行时长。

### 节点状态枚举

| 状态码 | 含义 |
|--------|------|
| 1 | 通过 |
| 0 | 拒绝 |
| 2 | 失败 |
| 3 | 其他（处理中等中间态） |

### 与其他表的关系

- 通过 `credit_sk` 关联 `dwd_fact_credit_application_di` 获取授信申请详情
- 通过 `node_id` 关联 `dim_node_decision_mapping_iri` 获取节点类型（决策前/中/后）
